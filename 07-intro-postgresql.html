Introducción a PostgreSQL

DBMS relacional avanzada y de código abierto.

Semestre 1, 2025

¿Qué es PostgreSQL?

  • Sistema de gestión de bases de datos relacional (RDBMS).
  • Software de código abierto con una comunidad activa.
  • Compatible con SQL estándar y con extensiones avanzadas.
  • Soporte para transacciones (Atomicity, Consistency, Isolation and Durability)

Historia y Popularidad

Historia y Popularidad

  • Iniciado en 1986 en Berkeley como parte del proyecto Ingres.
  • Se convirtió en PostgreSQL en 1996, con enfoque en extensibilidad.
  • Hoy en día es utilizado por empresas como Apple, Facebook y Spotify.
  • Reconocido por su estabilidad y capacidad de procesamiento de grandes volúmenes de datos.

Casos de Uso

  • Aplicaciones Web: PostgreSQL se usa en sistemas escalables con altos volúmenes de tráfico.
  • Análisis de Datos: Compatible con herramientas de BI y procesamiento masivo.
  • Sistemas Geoespaciales: A través de PostGIS, maneja datos espaciales para cartografía y análisis geográfico.
  • Banca y Finanzas: Ideal para sistemas transaccionales que requieren integridad y consistencia.

Arquitectura Cliente-Servidor

PostgreSQL sigue una arquitectura cliente-servidor que permite la escalabilidad.

  • Servidor de PostgreSQL:
    • Ejecuta procesos backend para gestionar consultas.
    • Utiliza buffers y caché para mejorar el rendimiento.
    • Maneja la autenticación y permisos de los clientes.
  • Cliente PostgreSQL:
    • Pueden ser interfaces gráficas (pgAdmin, DBeaver).
    • Pueden ser scripts o aplicaciones que se conectan vía librerías como psycopg2 (Python) o JDBC (Java).
    • Envía consultas SQL y recibe resultados del servidor.
  • Comunicación entre Cliente y Servidor:
    • Uso del protocolo PostgreSQL a través de TCP/IP.
    • Soporte para conexiones locales mediante sockets UNIX.
    • Implementación de SSL/TLS para seguridad en conexiones remotas.

Preparación para la Instalación

  • Sistemas Operativos Soportados: Windows, Linux, macOS.
  • Métodos de Instalación:
    • Paquetes oficiales para cada sistema operativo.
    • Uso de Docker para un entorno portátil y replicable.
    • Implementación en la nube con AWS RDS, Google Cloud SQL.
