Data Base Management System

DBMS

Semestre 1, 2025

¿Qué es un DBMS?

Un DBMS es un software que organiza, almacena y gestiona datos, permitiendo a los usuarios acceder, modificar y analizar información.

Ejemplo 1

Un banco utiliza un DBMS para gestionar cuentas y transacciones.

Ejemplo 2

Una red social organiza perfiles de usuario y publicaciones.

Clasificación por Modelo de Datos

Relacionales (RDBMS)

Datos organizados en tablas, usan SQL para consultas.

Ejemplo: MySQL, PostgreSQL.

NoSQL

Diseñados para datos no estructurados y escalabilidad horizontal.

Ejemplo: MongoDB, Neo4j.

Jerárquicos

Estructura tipo árbol.

Ejemplo: IBM IMS.

En Red

Relacionan datos mediante grafos.

Ejemplo: Integrated DBMS.

Relacionales vs. No Relacionales

Relacionales

  • Estructura fija con tablas.
  • Usan SQL como estándar.
  • Ideal para aplicaciones transaccionales.

No Relacionales

  • Flexibilidad para datos no estructurados.
  • Escalabilidad horizontal.
  • Ideal para big data y tiempo real.

Clasificación por Tipo de Uso

Centralizados

Toda la base de datos en un solo lugar.

Distribuidos

Datos replicados o fragmentados en múltiples ubicaciones.

Ejemplo: Amazon DynamoDB.

En Memoria

Datos almacenados en RAM para mayor velocidad.

Ejemplo: Redis.

Arquitecturas de DBMS

Monolítica

Todo integrado en una instancia.

Ejemplo: SQLite.

Cliente-Servidor

Separación entre cliente y almacenamiento.

Ejemplo: MySQL.

Multinivel

Grandes sistemas empresariales.

Ejemplo: Oracle.

¿Cómo Funciona un DBMS?

  1. El usuario envía una consulta.
  2. El DBMS la interpreta y planea su ejecución.
  1. El motor de almacenamiento accede a los datos requeridos.
  2. Se envían los resultados al usuario.

Almacenamiento de Datos

Tablas

Organizan datos en filas y columnas.

Índices

Mejoran la velocidad de búsqueda.

Espacios de Tablas

Tablespaces

  • Donde residen físicamente las tablas e índices.
  • Unidad lógica que agrupa uno o más archivos físicos.
  • Segregación / Seguridad.

Capas de Almacenamiento

Nivel Físico

Archivos y bloques en el disco.

Nivel Lógico

Tablas y relaciones.

Nivel Conceptual

La interfaz que perciben los usuarios.
