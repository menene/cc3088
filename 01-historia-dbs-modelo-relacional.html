Historia de las Bases de Datos

Semestre 1, 2025

¿Qué es la Información?

La información es un recurso estructurado y procesado a partir de datos, utilizado para tomar decisiones y resolver problemas.

Relación entre Conceptos

  • Datos: Elementos sin contexto (e.g., "123").
  • Información: Datos organizados (e.g., "123 representa un ID").
  • Conocimiento: Uso de la información para decisiones.

Ejemplo

En un hospital, los datos como nombre y edad se convierten en información al organizarse en historiales médicos. Que se convierten en conocimiento cuando se diagnostica.

Evolución de la Información

  • Pictogramas y registros manuales: Primeros sistemas para registrar datos.
  • Archivos computarizados: Años 60, almacenamiento secuencial en cintas.
  • Modelo relacional: Introducido en 1970, permitió almacenar datos en tablas estructuradas con relaciones claras.

Impacto

La evolución tecnológica ha permitido manejar grandes volúmenes de información de manera más eficiente.

¿Qué es un Sistema de Ficheros?

Un sistema de ficheros es un método utilizado por el sistema operativo para almacenar, organizar y acceder a datos en dispositivos de almacenamiento como discos duros, SSDs o cintas.

Componentes Principales

  • Archivos: Colecciones de datos relacionados.
  • Directorios: Estructuras jerárquicas que agrupan archivos.
  • Bloques de almacenamiento: Unidades físicas donde se guardan los datos.

Ejemplo Práctico

Un archivo de texto se guarda como una colección de datos en bloques, referenciados por su ruta en un directorio (e.g., C:/documentos/archivo.txt).

Ventajas de los Sistemas de Ficheros

  • Simplicidad: Fácil de implementar y comprender para sistemas pequeños.
  • Bajo costo: Ideal para dispositivos con recursos limitados.
  • Independencia: No necesita software adicional más allá del sistema operativo.

Limitaciones de los Sistemas de Ficheros

  • Redundancia de datos: Múltiples copias de la misma información pueden generar inconsistencias.
  • Falta de integridad: No hay mecanismos automáticos para garantizar la consistencia de los datos.
  • Dificultad en la recuperación de datos: Buscar datos específicos entre múltiples archivos puede ser complicado y lento.
  • Escalabilidad limitada: Diseñado para manejar cantidades pequeñas de datos.
  • Falta de seguridad: No existen controles avanzados de acceso ni auditorías.

Transición hacia las Bases de Datos

Problemas en los sistemas de ficheros llevaron a la creación de bases de datos relacionales:

  • Eliminan redundancias.
  • Garantizan integridad.
  • Facilitan consultas optimizadas.

Introducción al modelo relacional como solución.

Introducción a las Bases de Datos

¿Qué es una base de datos?

Una base de datos es una colección organizada de datos relacionados que permite un acceso eficiente y estructurado.

Beneficios

  • Reducción de redundancias.
  • Acceso rápido y eficiente a la información.
  • Mejora de la consistencia e integridad de los datos.

Conceptos básicos: Tablas, claves primarias y claves foráneas.

Historia del Modelo Relacional

Edgar F. Codd

Matemático británico, creador del modelo relacional en 1970.

Publicación del artículo: "A Relational Model of Data for Large Shared Data Banks".

Características Principales

  • Tablas como estructuras básicas para organizar datos.
  • Relaciones claras mediante claves primarias y foráneas.
  • Basado en la teoría de conjuntos y lógica de primer orden.

Problemas de los Sistemas Previos al Modelo Relacional

Sistemas jerárquicos

Estructura tipo árbol. Rigidez al manejar relaciones complejas. Ejemplo: IBM IMS.

Sistemas en red

Relacionaban datos mediante grafos. Más flexibles pero difíciles de consultar. Ejemplo: Integrated Database Management System (IDMS).

Evolución del Modelo Relacional

70's

IBM desarrolla el prototipo System R.

80's

Surgimiento de sistemas comerciales como Oracle Database y IBM DB2.

90's

Dominio del modelo relacional en la industria; expansión hacia sectores como finanzas, salud y comercio.

Impacto del Modelo Relacional

Adopción masiva

Presente en industrias como banca, salud, gobierno y educación.

Estandarización con SQL

Lenguaje universal para consultas relacionales.

Innovación continua

Sistemas híbridos como PostgreSQL que integran datos estructurados y semiestructurados.

Reconocimiento

Edgar Codd recibe el premio Turing en 1981.

Introducción a los DBMS Relacionales

Los DBMS relacionales son herramientas diseñadas para almacenar, organizar y consultar datos estructurados en tablas.

SQLite

DBMS ligero y embebido que no requiere configuración de servidor.

Ventajas

Libre, fácil de usar, ideal para apps móviles y prototipos.

Desventajas

No diseñado para aplicaciones con alta concurrencia.

Usos comúnes

Bases de datos locales en aplicaciones móviles.

MySql

Creado en 1995; adquirido por Oracle Corporation. Popular por su uso en aplicaciones web.

Ventajas

Amplio soporte, rendimiento alto, excelente documentación.

Desventajas

Algunas funciones avanzadas limitadas en la versión libre.

Usos comúnes

Aplicaciones móviles.

MariaDB

Fork de MySQL creado en 2009 por los desarrolladores originales.

Ventajas

Más rápido que MySQL, completamente libre.

Desventajas

Menor adopción en herramientas de terceros.

Usos comúnes

Sustitución directa de MySQL.

Microsoft SQL Server

DBMS desarrollado por Microsoft.

Ventajas

Integración robusta con herramientas como Power BI y Visual Studio. Soporte empresaria.

Desventajas

Licencias comerciales costosas.

Usos comúnes

Sistemas empresariales basados en Windows.

PostgreSQL

DBMS avanzado, conocido por su extensibilidad.

Ventajas

Soporte para JSON, vistas materializadas y funciones avanzadas, altamente extensible.

Desventajas

Curva de aprendizaje pronunciada y mayor consumo de recursos

Usos comúnes

Aplicaciones empresariales y científicas.
