Introducción a la Normalización de Bases de Datos

Semestre 1, 2025

¿Qué es la Normalización?

Proceso para organizar los datos reduciendo redundancias y mejorando la integridad.

¿Por qué es necesaria?

  • Evita datos duplicados
  • Minimiza inconsistencias
  • Facilita la actualización y modificación de datos
  • Optimiza almacenamiento y rendimiento

Problemas de Bases de Datos No Normalizadas

  • Redundancia de datos
  • Anomalías de inserción
  • Anomalías de eliminación
  • Anomalías de actualización

Teoría de Dependencias Funcionales

Dependencias Funcionales

Un atributo B de una relación depende funcionalmente de otro atributo A si a todo valor de A le corresponde siempre el mismo valor de B.

                        A -> B

En el contexto de bases de datos

Relación entre dos o más atributos de una tabla, donde el valor de uno de ellos determina el valor de los demás.

A también es llamada Determinante

B también es llamada Dependiente

Podemos pensar que el determinante es el id de una tabla.

Por ende el dependiente serían las demás columnas de la tabla.

id -> nombre

id -> email

id -> telefono

Tipos

Dependencia Funcional Completa

Un atributo depende completamente de la clave primaria.

(id_pedido, id_producto) → cantidad

Aquí, `cantidad` depende de ambos atributos, no de uno solo.

Dependencia Funcional Parcial

Un atributo depende solo de parte de la clave compuesta.

(id_pedido, id_producto) → nombre_producto

Aquí, `nombre_producto` depende solo de `id_producto`, no del pedido.

Dependencia Funcional Transitiva

Un atributo depende de otro atributo que no es clave.

id_cliente → id_ciudad → nombre_ciudad

Aquí, `id_ciudad` es dependiente de `id_cliente`, y `nombre_ciudad` depende de `id_ciudad`, lo que puede causar problemas.

Ventajas de Aplicar la Normalización

  • Evita la redundancia de datos
  • Asegura la integridad referencial
  • Facilita la actualización de datos
  • Optimiza el espacio de almacenamiento

Errores Comunes en Normalización

  • No identificar correctamente las dependencias funcionales
  • Eliminar redundancia sin considerar consultas futuras
  • Sobre-normalizar
  • Pensar que normalizar es siempre la mejor solución
